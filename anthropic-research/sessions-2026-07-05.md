## [Claude Code v2.1.200: Default Permission Mode Changed to "Manual", AskUserQuestion No Longer Auto-Continues](https://github.com/anthropics/claude-code/releases)
**Source**: Anthropic
**Type**: Product Update / Security Change
**Date**: 2026-07-03
**Links**: [Claude Code Changelog](https://code.claude.com/docs/en/changelog) · [GitHub Releases](https://github.com/anthropics/claude-code/releases) · [Releasebot tracker](https://releasebot.io/updates/anthropic/claude-code)

### Speaker / Author
Anthropic Editorial — released via the Claude Code changelog as version 2.1.200 on July 3, 2026. This is the first significant change to Claude Code's default behaviour mode since the introduction of the permission model in late 2025.

### Significance Rating
⭐⭐⭐☆☆ — A significant behaviour change that every Claude Code user will notice immediately: the default permission mode shifts from "default" (which allowed some autonomous actions) to "Manual" (which requires explicit approval for each tool call), and AskUserQuestion dialogs no longer auto-continue on a timer. This increases safety and user control at the cost of slightly higher friction for automated workflows that relied on the previous defaults.

### 摘要
Claude Code 2.1.200 将默认权限模式从"default"改为"Manual"，要求对每次工具调用明确确认；同时取消 AskUserQuestion 对话的超时自动继续，须手动操作。此外修复了后台会话在休眠/唤醒后停滞、速率限制导致子代理静默失败等多项稳定性问题。

### Summary
Claude Code version 2.1.200, released July 3, 2026, makes two headline behaviour changes. First, the **default permission mode is now "Manual"** across the CLI, VS Code extension, and JetBrains plugin — replacing the previous "default" mode that allowed Claude to proceed autonomously with certain tool calls without per-action confirmation. Developers who previously relied on a more permissive baseline will now see more approval dialogs. Second, **AskUserQuestion dialogs no longer auto-continue**: previously, an idle-timer would allow Claude to proceed if the user did not respond within a set window; now, the user must explicitly confirm every dialog. Auto-continuation can be re-enabled per-session via `/config askUserQuestion.idleTimeout=<seconds>`. Alongside these changes, the release also fixes: background sessions stalling after sleep/wake cycles, subagents silently returning empty results when cut off by rate limits (now they return partial work with a clear error), control-byte leakage from background-agent output to the terminal, and various screen-reader accessibility improvements.

### Main Takeaway
Claude Code's default posture is now more conservative — every tool call in Manual mode requires human sign-off, making it safer for enterprise deployments but more disruptive for developers who built unattended automation on the old defaults.

### Relevance to Sunny
⭐⭐⭐⭐☆ — High value and immediately actionable for Sunny's rollout. Three implications. First, **developer communication**: any developers currently using Claude Code with background agents or unattended CI pipelines on the old "default" mode will find their workflows require more interaction after updating to v2.1.200; Sunny should include a heads-up in her next internal comms before her developers auto-update. Second, **security posture**: from an FCA/PRA model risk perspective, the new Manual default is strictly better — it ensures that no tool call happens without explicit human authorisation, addressing the "approval fatigue" attack vector documented in the "How We Contain Claude" engineering post (captured 06-24). Third, **enterprise configuration**: for developers who legitimately need more autonomous behaviour (e.g. background CI agents), the new `/config` opt-in path is cleaner and more auditable than the previous implicit default — Sunny's IT team should document the approved set of timeout configurations for different use-case tiers (interactive, supervised-background, fully-autonomous CI).

### 摘要

### Summary
Anthropic's Frontier Red Team published Phase Two of Project Fetch on June 18, 2026, testing whether Claude Opus 4.7 could autonomously programme an off-the-shelf robotic quadruped (a "robodog") to perform a set of structured robotics tasks — the same tasks used in Phase One, where Anthropic employees using Claude Opus 4.1 as an assistant took weeks to complete. Opus 4.7 operating solo completed the Phase Two tasks approximately **20× faster** than the fastest human-plus-AI team from Phase One, while producing roughly **10× less code** (1,045 lines vs. ~10,309 lines for the best team). The tasks covered sensor interfacing, object detection, and environment navigation — core software engineering problems in a constrained domain. Critically, Opus 4.7 could identify the optimal approach among multiple options on the first try, without the iterative trial-and-error that characterised human attempts. However, the model still failed the "fetching" subtask — precisely guiding the robodog to push a beach ball back to a target area — because the motion-control accuracy required for physical execution could not be achieved through code generation alone. The team concluded that this is **not a robotics breakthrough** but rather evidence of Claude's expanding capability as a general-purpose software engineer in unfamiliar domains.

### Main Takeaway
Claude Opus 4.7 can out-code an expert human team 20× faster in a domain it was never trained on — but physical-world reliability remains a separate, unsolved problem.

Developer teams using Claude Code should set expectations accordingly — Claude Code accelerates writing, not necessarily debugging production systems with external dependencies.
