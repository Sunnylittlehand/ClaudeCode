## [Claude in Microsoft Foundry Is Now Generally Available](https://claude.com/blog/claude-in-microsoft-foundry)
**Source**: Anthropic / Microsoft Azure
**Type**: Product Launch / Enterprise Infrastructure Announcement
**Date**: 2026-06-29
**Links**: [Anthropic announcement](https://claude.com/blog/claude-in-microsoft-foundry) · [Microsoft Azure Blog](https://azure.microsoft.com/en-us/blog/claude-in-microsoft-foundry-is-now-generally-available/) · [Microsoft Learn deployment guide](https://learn.microsoft.com/en-us/azure/foundry/foundry-models/how-to/use-foundry-models-claude)

### Speaker / Author
Anthropic Editorial and Microsoft Azure. This is a joint announcement marking the General Availability of Claude models natively inside Microsoft's AI platform — meaning Claude is now procured, governed, and billed through Azure rather than via a separate Anthropic account.

### Significance Rating
⭐⭐⭐⭐☆ — Azure-native GA with enterprise-grade governance (Entra ID, RBAC, zero data retention, US data zone, MACC drawdown) is a major distribution milestone that removes the primary IT-procurement blocker for financial services teams already standardised on Azure.

### 摘要
Claude Opus 4.8 和 Haiku 4.5 正式在微软 Foundry（Azure AI 平台）全面上线，企业用户现可通过现有 Azure Entra ID 身份、RBAC 权限体系与 MACC 预算直接部署和计费，支持美国数据驻留区及零数据留存选项。基础设施为 NVIDIA GB300 Blackwell Ultra GPU。对于已有 Azure 企业协议的受监管机构而言，这是无需额外供应商合同即可接入 Claude 的重大进展。

### Summary
Claude Opus 4.8 and Claude Haiku 4.5 are now generally available in Microsoft Foundry on Azure, running on NVIDIA GB300 Blackwell Ultra GPUs. Enterprise teams can authenticate via Microsoft Entra ID, apply Azure role-based access controls, and select a US data zone for data residency requirements; a zero-data-retention option ensures prompts and completions are not retained by Anthropic after the API call. Billing is unified in Claude Consumption Units (CCU) on the Azure invoice, and eligible customers with a Microsoft Enterprise Agreement can draw down their existing MACC commitment. This brings Claude into the same procurement channel as the rest of Azure, eliminating the need for a separate Anthropic commercial agreement.

### Main Takeaway
Claude is now inside the Microsoft enterprise agreement — Azure-first financial-services teams can deploy it without a separate vendor contract.

### Relevance to Sunny
⭐⭐⭐⭐⭐ — Must-read for her rollout strategy. Most investment banks are standardised on Azure, meaning their IT, procurement, and InfoSec teams are already familiar with Microsoft's governance toolchain. Azure Entra ID authentication and RBAC mean Sunny can map Claude access to existing Active Directory groups and policies without building bespoke identity plumbing. The zero-data-retention option directly addresses FCA/PRA client-data and confidentiality concerns. MACC drawdown means Claude spend comes off an existing Azure budget line already approved through the bank's vendor management process — which could dramatically shorten procurement timelines compared to a standalone Anthropic contract. The US data zone is a start, but Sunny should check with her team whether Azure EU data zone support is on the roadmap for UK/EU regulatory compliance.
