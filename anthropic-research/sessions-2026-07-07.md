### Speaker / Author
The Mills Review was commissioned by the FCA and led by Baroness Sarah Mills, with evidence gathered from over 150 firms, consumer groups, academics, and regulators. The FCA (Financial Conduct Authority) is the UK's primary conduct regulator for financial services firms and its conclusions are binding context for any regulated institution deploying AI tools. ### 摘要
英国金融行为监管局（FCA）于2026年7月6日正式发布《米尔斯报告》，提出七项优先建议，为零售金融服务领域的人工智能监管制定2030年路线图。报告确认《消费者责任》为AI消费者保护的核心框架，高级经理制度（SM&CR）问责模式维持不变，但将出台更清晰的指导方针，明确在AI委托程度加深时高级经理须采取的合理步骤。报告特别关注"自主代理金融"（Agentic Finance）的兴起，强调须建立可信代理标准，并呼吁通过开放金融基础设施加以落实。报告不建议出台专门的AI立法，而是倡导对现有监管框架进行演进式调整。这是FCA迄今就金融服务领域AI使用发布的最重要政策文件。

### Summary
The FCA published the Mills Review on 6 July 2026, setting out seven priority recommendations to govern AI in UK retail financial services through 2030. The review confirms the Consumer Duty as the primary framework for AI consumer protection and leaves the SM&CR accountability model unchanged, but calls for clearer guidance on the reasonable steps senior managers must demonstrate as AI delegation deepens. A central theme is the rise of "agentic finance" — AI systems that can act autonomously within pre-set goals — with the report calling for trusted agent standards built through Open Finance infrastructure. Crucially, the review rejects AI-specific legislation in favour of evolving existing frameworks, giving firms more certainty but also more accountability for outcomes. The FCA's AI Lab will be scaled up significantly to support testing and innovation, and an "Agentic Supervisory Model" will be developed by the regulator itself.

### Main Takeaway
The FCA has drawn the line on AI governance in UK financial services: existing frameworks hold, Consumer Duty governs outcomes, and senior managers are accountable — but agentic AI now demands explicit pre-deployment assurance evidence.

## [Introducing the Claude Apps Gateway for Amazon Bedrock and Google Cloud](https://claude.com/blog/introducing-the-claude-apps-gateway)
**Source**: Anthropic
**Type**: Product Launch / Blog Post
**Date**: 29 June 2026
**Links**: [Official Anthropic blog](https://claude.com/blog/introducing-the-claude-apps-gateway) · [Claude Code docs](https://code.claude.com/docs/en/claude-apps-gateway) · [DevOps.com coverage](https://devops.com/anthropic-adds-enterprise-gateway-to-simplify-claude-code-access-on-aws-and-google-cloud/) · [Clauding.de deep-dive](https://clauding.de/en/posts/claude-apps-gateway-bedrock-google-cloud)

### Speaker / Author
Anthropic Editorial — the Claude Code platform team. The Claude Apps Gateway is a purpose-built enterprise infrastructure component, reflecting the broader trend of Anthropic investing heavily in the plumbing that lets regulated industries deploy Claude Code without granting developers direct API key access.

### Significance Rating
⭐⭐⭐⭐⭐ — Closes the single largest gap in enterprise Claude Code deployments: identity federation and spend control within customers' existing cloud infrastructure.

### 摘要
Anthropic于2026年6月29日发布Claude Apps Gateway，专为企业在Amazon Bedrock、Google Cloud或Microsoft Foundry上部署Claude Code而设计。该网关作为单一无状态容器运行于客户自有基础设施中，充当OpenID Connect（OIDC）依赖方，支持Google Workspace、Microsoft Entra ID及Okta等企业身份提供商。核心功能包括：按身份提供商组别进行RBAC权限控制（开发者无法在本地覆盖）、组织/团队/用户三级每日/每周/每月消费上限，以及零数据回传Anthropic的推理路由（除非主动配置Claude API）。该网关打包在现有claude可执行文件中，无需额外部署工具，可与PostgreSQL后端配合实现审计追踪。这是Anthropic迄今为止面向监管环境最完整的企业身份与治理解决方案。

### Summary
Anthropic launched the Claude Apps Gateway on 29 June 2026, a self-hosted control plane that lets enterprises run Claude Code through Amazon Bedrock, Google Cloud, or Microsoft Foundry with corporate identity federation, centralised policy enforcement, and per-user cost tracking. The gateway ships inside the existing `claude` binary (no new deployment tooling required) and runs as a stateless container backed by PostgreSQL. It acts as an OIDC relying party against any standards-compliant identity provider — Google Workspace, Microsoft Entra ID, or Okta — so developers authenticate with corporate credentials rather than holding API keys or cloud credentials. RBAC is enforced by IdP group rather than by individual developer, meaning different teams can be scoped to different models, tools, and permissions that a developer cannot override locally. Spend limits can be set per organisation, group, or user on daily, weekly, or monthly cycles with automatic enforcement. Inference traffic does not flow through Anthropic's infrastructure unless explicitly configured to use the Claude API — the gateway routes directly to Bedrock/GCP/Foundry endpoints within the customer's own cloud perimeter.

### Main Takeaway
Anthropic has shipped the enterprise identity and governance plumbing that makes Claude Code deployable inside regulated cloud environments without granting developers raw API key access.

### Relevance to Sunny
⭐⭐⭐⭐⭐ — Must-read — directly shapes rollout decisions. The Claude Apps Gateway addresses the three blockers most frequently cited by enterprise security teams when evaluating Claude Code: static credential management, inability to enforce model-level access controls by team, and lack of per-developer spend visibility. Running inside Bedrock or GCP within the bank's own cloud perimeter means inference traffic stays within the existing data zone boundary — directly relevant to any FCA data residency or DORA operational resilience requirements. The PostgreSQL audit backend gives Sunny's governance team an immutable log of who ran what model, when, and at what cost — exactly the auditability evidence SM&CR senior managers need to demonstrate reasonable steps. The zero-additional-binary approach (ships in the existing `claude` binary) also reduces the deployment approval surface.
