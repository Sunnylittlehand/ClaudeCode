## [OpenAI Retracts SWE-Bench Pro Recommendation After Audit Finds ~30% of Tasks Broken](https://openai.com/index/separating-signal-from-noise-coding-evaluations/)
**Source**: OpenAI Research
**Type**: Research / Benchmark Audit
**Date**: 2026-07-08
**Links**: [OpenAI research post](https://openai.com/index/separating-signal-from-noise-coding-evaluations/) · [OpenAI retracts recommendation](https://officechai.com/ai/openai-retracts-recommendation-to-use-swe-bench-pro-as-coding-eval-over-30-broken-tasks/)

### Speaker / Author
OpenAI Research team. Follows an earlier audit of SWE-bench Verified that similarly found fundamental issues. OpenAI is now the dominant critic of its own previously-recommended coding benchmarks — an unusual but credibility-building move.

### Significance Rating
⭐⭐⭐☆☆ — Benchmark reliability is foundational to AI capability comparisons; a 30% broken task rate means the leaderboard on which competitive coding claims rest is materially unreliable.

### 摘要
OpenAI审计SWE-Bench Pro后发现约30%的任务存在设计缺陷（隐含需求、矛盾指令、测试过严等），不再推荐将其用于前沿编程能力评估。这对依赖基准测试做产品决策的企业买家具有重要警示意义。

### Summary
OpenAI audited the 731-task public split of SWE-Bench Pro using model-based investigator agents and five independent software engineers, finding that roughly 27–34% of tasks are broken — containing hidden requirements, overly strict tests, or contradictory instructions that cause functionally correct solutions to fail. OpenAI has retracted its recommendation to use SWE-Bench Pro as a measure of frontier coding capability. Both SWE-bench Verified (the predecessor) and SWE-Bench Pro (the intended replacement) have now been found unreliable, leaving the coding evaluation space without a broadly trusted standard benchmark.

### Main Takeaway
The benchmark scores used to compare Claude Code vs Codex are built on contested foundations — enterprise buyers should weight real-task evaluations over leaderboard positions.

### Relevance to Sunny

Anthropic Claude Code Engineering team. The in-app browser shipped in v2.1.206 and was the headline feature of the Week 28 digest published July 10.

### Significance Rating
⭐⭐⭐☆☆ — Meaningfully reduces context-switching friction for developers and closes a significant UX gap with Codex's desktop environment.

### 摘要
Claude Code桌面版（v2.1.206）内置沙盒浏览器，可直接访问文档、设计稿等外网页面，与本地开发服务器操作方式一致。使用独立浏览配置，不共享个人账户或历史记录，快捷键Cmd/Ctrl+Shift+B唤起。

### Summary
Claude Code Desktop's Week 28 release (v2.1.206, July 10) adds a built-in Browser panel: a sandboxed tabbed browser Claude drives the same way it handles local dev-server previews — opening documentation, design mockups, or any external site, reading content, and clicking through flows. The browser runs a clean, isolated profile (no personal logins, no browsing history) and is configured separately from the user's personal browser. Keyboard shortcut is Cmd+Shift+B (macOS) or Ctrl+Shift+B (Windows). It differs from the Claude in Chrome extension: the in-app browser is for clean-profile work; the Chrome extension shares the user's existing login sessions.

### Main Takeaway
Claude Code can now reference external documentation and live sites without developers switching windows — reducing friction on multi-step agentic tasks.

### Relevance to Sunny
⭐⭐⭐☆☆ — Practical improvement for developer teams in her rollout: agents can now pull API documentation, internal wiki pages (if accessible), or design specifications directly within the coding session.
