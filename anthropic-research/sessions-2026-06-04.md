### Summary
Anthropic's second annual Code with Claude developer conference (San Francisco, May 6) anchored a global tour (London May 19, Tokyo June 10) and delivered the most consequential set of Claude Code enterprise announcements to date. The headline compute deal with SpaceX gave Anthropic exclusive access to the Colossus 1 supercluster — over 220,000 NVIDIA H100/H200/GB200 GPUs and 300 MW of capacity — immediately doubling five-hour rate limits for Pro, Max, Team, and Enterprise plan users and removing peak-hour caps. The Dynamic Workflows feature (research preview in CLI, Desktop, and VS Code for Max/Team/Enterprise) lets Claude generate its own orchestration scripts that fan work out across tens to hundreds of specialised subagents, enabling parallel inspection of large codebases and automated validation loops. Claude Managed Agents launched with three pillars: Multiagent Orchestration (fleet-scale task decomposition), Outcomes (define success criteria and let agents iterate), and Dreaming (cross-session memory so agents build on past work). Claude Security entered public beta for Enterprise customers, allowing security teams to scan code for vulnerabilities and generate proposed fixes using Claude. Self-hosted sandboxes moved to public beta, allowing enterprises to route tool execution through their own infrastructure or managed providers (Cloudflare, Daytona, Modal, Vercel) and connect to private MCP servers. Enterprise Admin Controls were extended with Connector Permissions, letting admins control which connectors and tools each custom role can access.

### Main Takeaway
Claude Code has crossed from developer productivity tool to enterprise agentic platform in a single event — compute abundance, security scanning, and governance controls are now shipping together.
