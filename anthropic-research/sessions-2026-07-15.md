## [Introducing Claude for Teachers — Free Premium Claude for US K-12 Educators](https://www.anthropic.com/news/claude-for-teachers)

Anthropic Editorial. Developed in partnership with the **American Federation of Teachers (AFT)** and with learning scientists embedded in the product design. ### Significance Rating
⭐⭐⭐⭐ — A substantive new product launch that extends Claude's enterprise compliance and data-governance frameworks into a new regulated sector (K-12 education) and signals Anthropic's strategy for building institutional trust sector by sector ahead of its IPO.

### 摘要
Anthropic 于 7 月 14 日推出 Claude for Teachers，向美国经过核实的 K-12 教师免费提供一年 Claude 高级权限，包含 Claude Code、Claude Cowork、覆盖全美 50 州学术标准的 Learning Commons 连接器及九项教育工具集成。数据不用于模型训练，并依据 FERPA 签订 K-12 数据处理协议，由美国教师联合会（AFT）共同制定隐私条款。注册截止至 2027 年 6 月 30 日，学校及学区版本另行推出。

### Summary

The competitive context is explicit — OpenAI already has education programmes and Google has deep classroom incumbency, making this a direct market-expansion move.

### Main Takeaway
Anthropic is using Claude for Teachers to build sector-by-sector institutional trust through compliance-first design — the FERPA/AFT terms are the product, not a footnote.

## [Claude Code v2.1.210 (Week 31): Subagent Worktree Isolation Fix, Prompt-Injection Hardening, 30+ Bug Fixes](https://code.claude.com/docs/en/changelog)
**Source**: Anthropic
**Type**: Product update / Changelog
**Date**: 2026-07-14
**Links**: [Claude Code changelog](https://code.claude.com/docs/en/changelog) · [GitHub releases](https://github.com/anthropics/claude-code/releases)

### Speaker / Author
Anthropic Engineering. Version 2.1.210 is the primary Week 31 release, accompanied by the minor fix 2.1.209 (reverting an overly broad guard blocking `/model` and other dialogs in background sessions).

### Significance Rating
⭐⭐⭐ — A substantial maintenance and security release; two fixes (subagent worktree isolation and indirect prompt-injection hardening) are directly material to enterprise deployments in regulated environments.

### 摘要
Claude Code v2.1.210 修复了超过 30 个漏洞，包含两项关键安全修复：子代理 worktree 隔离漏洞（子代理可对主仓库执行 git 破坏性命令）及 Agent 工具间接提示注入防护加强。另新增实时工具耗时计数器、权限规则启动警告，并改进自动模式默认为 Sonnet 5（外部会话中经验证后锁定）。

### Summary
Claude Code v2.1.210 shipped July 14 with over 30 bug fixes and several new features. Two security fixes are material for enterprise environments: (1) a fix for `isolation: 'worktree'` subagents being able to run git-mutating commands against the main repo checkout instead of their own isolated worktree — meaning prior versions could allow a subagent to corrupt a developer's main branch; and (2) hardening of the Agent tool against indirect prompt injection via content a subagent read — a vector where malicious content in a code file or document could manipulate Claude's behaviour. Additional changes include a live elapsed-time counter on collapsed tool summaries (visibility improvement for long-running agents), a startup warning for `Write(path)`, `NotebookEdit(path)`, and `Glob(path)` permission rules (recommending safer `Edit(path)` / `Read(path)` equivalents), and improved auto mode defaulting to Sonnet 5 for external sessions with session-pinning on first validation. A Fable known issue is noted: Fable temporarily shows as unavailable in the advisor picker while a server-side issue is fixed.

### Main Takeaway
Two security-class bugs in Claude Code — subagent worktree isolation and prompt-injection via read content — are now fixed; enterprise teams should verify they are on v2.1.210+.

### Relevance to Sunny
⭐⭐⭐⭐ — The worktree isolation fix is directly relevant to Sunny's rollout governance. If a bank developer was running Claude Code agents on a production-adjacent codebase before this patch, a subagent could have made git mutations to the main repo checkout rather than its isolated branch — a code-integrity risk with potential audit and change-control implications under the bank's SDLC policies. Sunny should confirm all Claude Code installations are on v2.1.210+ and add a minimum-version check to the IT policy for the rollout. The prompt-injection hardening is equally important: in a codebase where third-party dependencies or external API responses could contain adversarial text, prior versions were exploitable via the Agent tool. This fix should be a mandatory patch in any regulated-environment deployment.
