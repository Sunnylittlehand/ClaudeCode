## [Claude Workload Identity Federation (WIF) Now Generally Available](https://claude.com/blog/workload-identity-federation)
**Source**: Anthropic
**Type**: Product announcement / enterprise security
**Date**: 2026-06-19
**Links**: [Anthropic blog](https://claude.com/blog/workload-identity-federation) · [API Docs](https://platform.claude.com/docs/en/manage-claude/workload-identity-federation) · [Security Boulevard analysis](https://securityboulevard.com/2026/06/anthropic-workload-identity-federation-what-it-gets-right-and-what-it-still-doesnt-solve/)

### Speaker / Author
Anthropic Editorial — published by the Claude Platform team. The announcement went live June 19, 2026 on the Claude developer blog.

### Significance Rating
⭐⭐⭐⭐ — Eliminates long-lived static API keys for Claude integrations and replaces them with short-lived OIDC tokens; a meaningful step-change in enterprise security posture for regulated deployments.

### 摘要
Anthropic正式发布Workload Identity Federation（WIF）功能，彻底告别静态API密钥。企业工作负载可通过AWS IAM角色、Azure托管身份、GCP服务账户、GitHub Actions令牌、Kubernetes服务账户或Okta等OIDC兼容提供商进行身份验证，获取按需签发的短效访问令牌。每个工作负载拥有独立的服务账户、专属的权限范围和完整的审计追踪，显著降低密钥泄露风险，满足金融监管对凭证管理的严格要求。

### Summary
Anthropic's Workload Identity Federation, now generally available, replaces static `sk-ant-` API keys with short-lived OIDC tokens issued against your existing identity provider — AWS IAM roles, Azure managed identities, GCP or Kubernetes service accounts, GitHub Actions tokens, Okta, or any OIDC-compliant system. Each workload gets its own service account with scoped permissions and a separate audit trail instead of a shared key. Tokens expire in minutes rather than never, dramatically reducing the blast radius of any credential exposure. The docs note that federation security is only as strong as the upstream IdP configuration, so well-scoped service accounts are still required.

### Main Takeaway
Anthropic has removed the single biggest credential-management blocker for enterprise Claude deployments — no static keys to create, rotate, leak, or share.

### Relevance to Sunny
⭐⭐⭐⭐⭐ — **Must-read for Sunny's rollout.** Static API keys are almost certainly on the FCA/PRA's list of unacceptable credential management practices for production AI systems. WIF solves this: per-workload service accounts with expiring tokens, existing-IdP integration (likely Azure AD / Entra ID in a UK bank), and a per-workload audit trail that maps directly to model risk management and auditability requirements. This should be surfaced immediately to the bank's CISO and IT security team as a blocker-clearing feature. Caveat: Anthropic's docs warn that WIF is only as strong as IdP configuration — security teams will need to enforce least-privilege OIDC scopes.

## [Claude Managed Agents: Self-Hosted Sandboxes and Private MCP Tunnels](https://claude.com/blog/claude-managed-agents-updates)
**Source**: Anthropic
**Type**: Product announcement / enterprise infrastructure
**Date**: 2026-05 (announced at Code with Claude London; blog post May/June 2026)
**Links**: [Anthropic blog](https://claude.com/blog/claude-managed-agents-updates) · [The Decoder](https://the-decoder.com/anthropic-adds-self-hosted-sandboxes-and-mcp-tunnels-to-claude-managed-agents/) · [InfoQ](https://www.infoq.com/news/2026/05/claude-mcp-tunnels/) · [The New Stack](https://thenewstack.io/anthropic-mcp-tunnels-sandboxes/)

### Speaker / Author
Anthropic Editorial — Claude Managed Agents team. First announced at the Code with Claude London conference (May 2026); blog post and public beta released shortly after.

### Significance Rating
⭐⭐⭐⭐ — Allows enterprises to run Claude agent tool execution inside their own infrastructure perimeter, directly addressing the data residency and network-isolation concerns that block regulated-industry adoption.

### 摘要
Anthropic为Claude Managed Agents推出自托管沙箱（公测）和MCP隧道（研究预览）功能。企业可将工具执行环境迁移至自有基础设施或Cloudflare、Daytona、Modal、Vercel等托管服务提供商，同时Anthropic继续负责Agent循环的编排、上下文管理和错误恢复。MCP隧道则允许Agent通过加密出站连接访问内部MCP服务器，无需将这些服务器暴露于公网。敏感文件、代码库和内部系统全程保留在企业网络边界内，满足金融监管对数据驻留的严格要求。

### Summary
Anthropic's self-hosted sandboxes (public beta) move Claude's tool execution environment — file access, code execution, MCP server calls — to infrastructure the enterprise controls, while Anthropic's infrastructure retains the orchestration and agent loop. Supported sandbox providers include Cloudflare, Daytona, Modal, and Vercel, or the enterprise's own servers. MCP tunnels (research preview) let Managed Agents reach private internal MCP servers via an outbound encrypted connection, keeping those servers fully off the public internet. The net effect: sensitive repositories, files, and services never leave the enterprise network, while existing network policies, audit logging, and security tooling remain in force.

### Main Takeaway
Claude agents can now run inside a bank's security perimeter — data stays in-house, the bank's existing security tooling applies, and no internal systems are exposed to the internet.

### Relevance to Sunny
⭐⭐⭐⭐⭐ — **This is the architecture that makes Claude Code viable in a regulated bank.** The two most common IT security objections to agentic AI are (1) "code and data leave our perimeter" and (2) "we can't connect internal systems without exposing them." Self-hosted sandboxes and MCP tunnels answer both objections directly. Sunny should brief her CISO on this as a risk-control enabler, not just a developer feature, and push to get the bank into the self-hosted sandbox beta. The combination of WIF (keyless auth) + self-hosted sandboxes + MCP tunnels is now a credible enterprise-grade security stack.
