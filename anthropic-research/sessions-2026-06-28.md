### Significance Rating
⭐⭐⭐⭐☆ — Three materially new enterprise controls in a single week: CLI-based MCP OAuth authentication, sandbox credential isolation, and surfaced permission prompts from background subagents — each addressing a specific enterprise security gap.

### 摘要
Claude Code第26周发布三项企业安全新特性：命令行MCP OAuth登录/登出（`claude mcp login/logout`）、沙箱凭证文件隔离（防止子代理窃取API密钥），以及后台子代理权限提示可见化（不再静默拒绝）。另有37%的流式传输CPU占用优化及OpenTelemetry响应日志事件。这是近几周合规价值最高的一次发布。

### Summary
Claude Code Week 26 (June 22–26) ships three enterprise-relevant security controls. First, `claude mcp login <name>` and `claude mcp logout <name>` bring OAuth 2.1 authentication to MCP server connections directly from the CLI — including a `--no-browser` flag for SSH environments — eliminating manual credential handling. Second, a new `sandbox.credentials` setting blocks sandboxed commands from reading credential files and secret environment variables, preventing accidental or malicious exfiltration of API keys through subagent tool calls. Third, background subagents that previously auto-denied permission requests now surface those prompts in the main session UI — a developer can approve or deny without losing context, with `Esc` denying only the requesting agent. Secondary additions include org-configured model restrictions enforced across all model-picker surfaces, a new `autoMode.classifyAllShell` setting routing all Bash/PowerShell through the classifier, OpenTelemetry `claude_code.assistant_response` logging (redacted by default), auto-mode denial reasons in the transcript, and a 37% CPU reduction during streaming.

### Main Takeaway
Week 26 closes three specific enterprise security gaps: credential handling in MCP connections, credential leakage through sandboxed tools, and blind permission escalation from background agents.

### Summary
Claude Code Week 25 (June 15–19) delivers three additions. First, **Artifacts** reach formal beta on Team and Enterprise plans: live, interactive pages Claude Code publishes to private URLs on claude.ai, updating in place as sessions progress — use cases include annotated PR walkthroughs and session-data dashboards. Second, **tool parameter matching** in deny/ask permission rules allows policies targeting tools by input value: `Agent(model:opus)` blocks Opus-model subagent spawns while permitting Haiku/Sonnet; `Agent(isolation:*)` matches any explicitly-isolated spawn. ### Main Takeaway
