## [Claude Code Week 33 (v2.1.212): /fork Background Sessions, /subtask, and Runaway-Loop Safeguards](https://code.claude.com/docs/en/changelog)
**Source**: Anthropic
**Type**: Product Update / Changelog
**Date**: 2026-07-17
**Links**: [Changelog](https://code.claude.com/docs/en/changelog) · [GitHub Releases](https://github.com/anthropics/claude-code/releases)

### Speaker / Author
Anthropic Engineering. Week 33 release of Claude Code, continuing the weekly cadence of agentic workflow improvements and safety hardening. This release (48 changes) is meaningfully larger than a typical weekly patch.

### Significance Rating
⭐⭐⭐ — Important agentic workflow restructuring (/fork and /subtask) and the first explicit runaway-loop safeguards (WebSearch and subagent spawn caps) — relevant for any deployment relying on multi-agent or research-intensive workflows.

### 摘要
Claude Code v2.1.212 重构了子会话工作流：`/fork` 现在将对话复制到后台独立会话继续运行（原 in-session 子 Agent 功能改由 `/subtask` 承接）；新增每会话 WebSearch 工具调用上限（默认 200 次）和子 Agent 生成上限（默认 200 个），防止失控循环；MCP 工具调用超过 2 分钟自动后台化；修复多处内存泄漏（MCP stdio 服务器每个泄漏最高 64MB、LSP 文档无限增长等），并修复 Windows Group Policy 阻止 PowerShell 5.1 时后台 Agent 启动失败的问题。

### Summary
Claude Code v2.1.212 ships three meaningful operational changes. First, **workflow restructuring**: `/fork` now copies the current conversation into a new background session (appearing as its own row in `claude agents`) while the current session continues — the previous in-session subagent behaviour is now `/subtask`. Second, **runaway-loop safeguards**: a session-wide cap on WebSearch tool calls (default 200, tunable via `CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION`) to prevent search loops; and a per-session cap on subagent spawns (default 200, override with `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`) to prevent runaway delegation. MCP tool calls running longer than 2 minutes now automatically move to the background so the session stays usable. Third, **memory leak fixes**: MCP stdio server stderr accumulating up to 64 MB per server (now bounded); LSP documents staying open indefinitely (now LRU with 50-doc cap); async hook output retained after backgrounding; and unbounded growth in headless/SDK sessions from large tool-result payloads. Windows Group Policy blocking PowerShell 5.1 is now handled gracefully (daemon prefers PowerShell 7). Background agent result reporting improved — Claude now reports still-running agents and waits for real completion rather than fabricating results.

### Main Takeaway
The runaway-loop safeguards (WebSearch and subagent caps) are the most operationally significant change for enterprise deployments: uncontrolled recursion in agentic workflows is one of the least-visible and most expensive failure modes, and these caps put a governed ceiling on it.

### Relevance to Sunny
⭐⭐⭐ — Worth noting for developer team guidance. The WebSearch and subagent spawn caps are directly relevant to compliance: in regulated environments, unbounded tool-call loops are both a cost-control risk and an audit-trail problem (a session that spawns 10,000 web searches before it's noticed will be extremely difficult to review). The default caps of 200 are sensible starting points; Sunny should discuss with development teams whether lower limits are appropriate for their use cases. The /fork→background and /subtask restructuring will require updating any internal runbooks or Claude Code usage guides that reference the previous /fork behaviour.
